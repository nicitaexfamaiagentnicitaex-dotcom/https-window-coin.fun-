# $WINDOW Deployment Guide

Этот проект — статический сайт. Ниже описаны шаги, как задеплоить его бесплатно и привязать домен `window-coin.fun`.

---

## 1. Структура проекта

- `index.html`, `styles.css`, `script.js`, `fortunes.js` — основной фронтенд.
- `token-config.js` — **единственное место**, где задаётся токен (значение `window.WINDOW_TOKEN_VALUE`).
- `ChatGPT_Image_10_._2025_._05_56_00.png` — фон сайта.
- `window.png` — изображение логотипа.

При изменении файлов убедитесь, что относительные пути к изображениям остаются актуальными.

---

## 2. Деплой на GitHub Pages (бесплатно)

1. **Создайте репозиторий** на GitHub и загрузите все файлы проекта в корень.
2. В настройках репозитория откройте вкладку **Settings → Pages**.
3. В разделе **Build and deployment** выберите:
   - Source: `Deploy from a branch`
   - Branch: `main` (или тот, где лежит сайт), папка `/ (root)`
4. Через минуту-другую сайт будет доступен по адресу вида `https://<username>.github.io/<repo>/`.

---

## 3. Привязка домена `window-coin.fun`

### 3.1 Файл `CNAME`

В корне репозитория создайте файл `CNAME` (без расширения) с единственной строкой:

```
window-coin.fun
```

GitHub Pages автоматически привяжет домен и выпустит SSL-сертификат.

### 3.2 DNS-настройки у регистратора

Зайдите в панель управления доменом и пропишите записи:

| Запись | Хост | Значение                      | TTL |
|--------|------|--------------------------------|-----|
| A      | @    | 185.199.108.153                | 1h  |
| A      | @    | 185.199.109.153                | 1h  |
| A      | @    | 185.199.110.153                | 1h  |
| A      | @    | 185.199.111.153                | 1h  |
| CNAME  | www  | `<username>.github.io.`        | 1h  |

> Замените `<username>` на ваш GitHub-логин. Точки в конце оставляем — так корректно для CNAME.

После сохранения подождите 10–30 минут (иногда до 24 часов), пока обновятся DNS-записи. Проверить можно командой `nslookup window-coin.fun`.

---

## 4. Проверка

1. Откройте `https://window-coin.fun` и убедитесь, что сайт загружается.
2. Проверьте HTTPS: браузер должен показывать зелёный замок.
3. Если домен не открывается:
   - Убедитесь, что файл `CNAME` действительно в репозитории.
   - Проверьте, корректно ли прописаны DNS-записи.
   - Подождите до суток — иногда задержка у DNS.

---

## 5. Обновление токена

Чтобы изменить отображаемый и копируемый токен:

1. Откройте `token-config.js`.
2. Измените значение константы `TOKEN_VALUE`.  
3. Задеплойте изменения (push в GitHub). Через несколько секунд GitHub Pages обновит продакшн.

---

## 6. Альтернативный хостинг (по желанию)

- **Cloudflare Pages** или **Vercel** — также бесплатны и дают встроенный CDN. Процесс подключения домена аналогичный (CNAME/ALIAS на их адреса).
